Vargas::GraphManager Class Reference

#include <gdef.h>

Detailed Description

Create and manage a graph and specified subgraphs.

Given a reference sequence and a variant call file, a GDEF file uniquely defines a set of subgraphs deriving from the base graph. Subgraphs are generated by filtering the base graph based on the number of samples in the graph.
Subgraphs are generated from parent graphs, with the top level base graph implied as 100%. ':' indicates a child graph, '~' indicates the complement population. Multiple definitions are delimited with ';', and all labels must be defined before being referenced. For example:

1 "a=10" # Choose 10 samples from the base graph
2 "b=5%" # Choose 5% of the samples in the base graph
3 "a:c=5" # Choose 5 of the 10 samples in 'a'
4 "~a:d=5" # Choose 5 samples from the complement population of 'a'
5 "a:~e=5" # Invalid, complement graphs are implicit and cannot be directly defined.

The assignment can also be suffixed with 't', to select the top n SNPs from a KSNP file.

1 "a=10t" # Use the top 10 SNPs (KSNP file) OR first 10 haplotypes (VCF File)

The following labels are reserved:

  • BASE: Full graph
  • REF: Linear reference graph
  • MAXAF: Linear graph with the maximum allele frequency


GDEF file format:

1 @gdef[:TYPE]
2 ref=REFERENCE,var=VCF/KSNP,reg=REGION,nlen=NODE_LEN
3 label=POP_FILTER
4 ...
5 label=POP_FILTER


Where Type indicates the graph source type. Default "VCF". The other valid option is "KSNP". When in VCF mode, the filter corresponds to the sample and genotype. When in KSNP mode, the filter corresponds to the n'th SNP in the ksnp file. As a result, the node population for a SNP will have a size of the number of lines in the KSNP file, with 1 bit set. Usage:

#include <gdef.h>
int main() {
Vargas::GraphManager gm();
gm.write("ref.fasta", "variants.vcf", "x:0-100", "a=50;a:b=10;~a:c=10%", 1000, "out.gdf");
gm.load("out.gdf");
gm.base(); // shared_ptr to whole graph
gm.subgraph("a"); // 'a' subgraph
gm.subgraph("~a"); // complement of 'a' subgraph
gm.subgraph("c"); // Invalid
gm.subgraph("~a:c") // valid
gm.subgraph("a:REF") // Only reference alleles from subgraph a
}

There are two versions to obtain graphs. The version prefixed with "make_" will create the graph if it has not been made yet, gaurded with #pragma omp critical. The const versions subgraph() and base() will return a nullptr if the graph does not exist.
